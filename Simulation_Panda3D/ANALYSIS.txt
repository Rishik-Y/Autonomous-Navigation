Simulation_Panda3D Port Analysis
================================

Scope
-----
This document inventories the original Simulation code and maps the 2D Pygame system to the new Panda3D rendering/runtime architecture while preserving algorithmic behavior.

2) Graphics Strategy (Pygame -> Panda3D)
----------------------------------------

Old graphics model
- Immediate-mode 2D drawing per frame via pygame.draw and blits.
- Screen transforms handled by scale/pan conversion helpers.
- Roads/nodes/paths rendered in flat 2D.

New graphics model
- Use MAP_Panda3D primitives directly:
  - SceneRenderer for terrain/grid/road/node draw calls.
  - CameraController for live navigation.
  - Heightmap for elevation queries.
- graphics.py rewritten as SimulationGraphics:
  - Build spline list from map_data.VISUAL_ROAD_CHAINS + generate_curvy_path_from_nodes.
  - draw_static_scene(): renderer.draw_grid + draw_roads + draw_nodes.
  - draw_active_paths(): draw selected vehicle path as elevated 3D line via renderer.draw_path_lines.
- Result: all former line/circle/rect calls are replaced with SceneRenderer mesh/line/node drawing.

3) 3D Translation Plan (Truck Pose Mapping)
-------------------------------------------

2D control state retained
- MPC and planners remain in XY plane exactly as before.
- Car physics state remains x_m, y_m, angle (radians yaw in planar frame).

3D projection
- Each truck owns a Panda3D NodePath root + rgbCube visual child + CollisionBox child.
- Visual cuboid dimensions use config constants:
  - length = CAR_LENGTH_M, width = CAR_WIDTH_M, height = CAR_HEIGHT_M.
- Bottom-center pivot rule:
  - model origin kept at cuboid center, visual shifted +CAR_HEIGHT_M/2 in local Z.
  - truck node placed at ground-contact point Z.
- Per-tick projection sequence:
  1) MPC/physics updates x_m/y_m/angle.
  2) z_ground = Heightmap.get_height_at_world(x_m, y_m).
  3) node.setPos(x_m, y_m, z_ground).
  4) node.setH(degrees(angle)); P=0, R=0.
- This keeps algorithm math 2D while trucks visibly climb/descend terrain.

4) UI Plan (Tooltip Overlay in Panda3D)
---------------------------------------

Old UI
- Pygame tooltip near mouse cursor using translucent rect + rendered text.

New UI
- tooltip_overlay.py provides TooltipOverlay with OnscreenText on aspect2d.
- Interaction flow:
  - Primary: collision ray intersects truck CollisionBox (truck_id tag).
  - Secondary: ground pick + nearest zone lookup for load/dump nodes.
- Placement strategy:
  - Convert world XYZ (truck top or zone marker) to screen/aspect coordinates using camLens.project.
  - Place text above projected position.
- Content parity maintained:
  - Truck: id, state, target, cargo mass.
  - Mine: coal remaining, en_route.
  - Dump: total dumped, en_route.

5) Porting Decisions Applied
----------------------------
- Algorithm, MPC, dispatcher, utility logic copied and preserved.
- Pygame-dependent files fully rewritten for Panda3D runtime:
  - main.py
  - graphics.py
  - car.py
  - tooltip_overlay.py
- map_loader updated to resolve artifacts from MAP_Panda3D/Saved_Map first, with compatibility fallbacks.
- GraphAdapter defaults updated to read map_cache.pkl and waypoints.pkl via the same Saved_Map resolver.

6) Integration Contract with MAP_Panda3D
----------------------------------------
- Directly reuses:
  - panda_common.SceneRenderer
  - panda_common.CameraController
  - panda_common.Picker
  - panda_elevation.Heightmap
- Runtime expects map_data.json, mine_config.json, map_cache.pkl, waypoints.pkl from MAP_Panda3D/Saved_Map.
- Fallbacks retained for backward compatibility with existing Simulation/Map and MAP/Saved_Map artifacts.
