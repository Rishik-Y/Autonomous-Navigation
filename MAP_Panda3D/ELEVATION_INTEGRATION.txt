Elevation system components to import from Elevation
====================================================
Heightmap
- Integer cell grid aligned with MAP coordinates: origin(-650,-450), cell_size=10.0, cols=150, rows=140.
- Cell APIs, vertex interpolation, world/cell conversion, and JSON persistence format.

TerrainMesh
- Shared-vertex smooth terrain mesh built from Heightmap.
- GeomVertexData/GeomTriangles rendering with finite-difference normals and grey coloring by height.

CellHighlight (optional editor UX)
- Terrain cell overlay for terraforming feedback.

Collision-ray picking
- Camera ray against terrain geometry for accurate (x,y,z) surface picks.

Coordinate alignment
- Elevation and map coordinates match, so map XY can be rendered directly with terrain-derived Z.

Rendering pipeline
==================
1) Load heightmap grid (or default zeros if no saved elevation file).
2) Build TerrainMesh shared-vertex geometry from Heightmap vertex interpolation.
3) Render terrain as base layer with lighting.
4) For every map visual point (node/spline/route/grid sample):
   z = heightmap.get_height_at_world(x, y)
   render_z = z + small_visual_offset
5) Roads/nodes/waypoints therefore follow terrain profile while map logic remains XY-only.
6) Node labels anchor above elevation-adjusted positions.


Data flow and persistence
=========================
Input load order
- Elevation: Heightmap.load_json() from Elevation/Saved_Elevation/elevation_data.json (if present).
- Map artifacts: existing resolve_input_path behavior (Saved_Map preferred, then legacy/simulation fallbacks).

Runtime sharing
- Launcher creates one Heightmap/TerrainMesh/SceneRenderer/Picker used by all 5 modes.
- Modes continue editing/viewing map structures in XY.
- Renderer computes Z lazily at draw time from shared Heightmap.

Output compatibility
- map_data.py/json, map_cache.pkl, waypoints.pkl, mine_config.json unchanged in schema/path behavior.
- Simulation consumers keep reading the same structures; elevation affects visualization only.

Fallback behavior
- Missing elevation JSON => default zero grid, yielding flat terrain but same pipeline.
